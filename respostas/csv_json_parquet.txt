1 - Organize os arquivos brutos de forma livre (ex.: diretórios locais, diferentes formatos de 
arquivo). 
2 - Explique as vantagens de formatos como CSV, JSON e Parquet e justifique sua escolha. 
3 - Explique também como faria essa mesma organização em um bucket AWS S3.

Respostas:

1 - Organizei os meus arquivos que vão para a camada raw dentro da pasta 'inputs', com subdivisão de formato.

2 - A vantagens de se usar dados nos formatos listados é que são formatos leves e compactos, podendo conter milhares de registros sem pesar no processamento.

Para extair os dados escolhi os formatos json e csv, porque os dois conseguem manter a estrutura original vinda da requisição. Assim consigo trazer os dados conservados e da forma mais completa. 

Já na camada curated irei utilizar os dados em parquet, pois como as clouds cobram por quantidade de dados processados o parquet é colunar e compacto e agiliza as querys.


3 - Para a estruturação da camada Raw no AWS S3, os dados seriam adicionados ao bucket no formato bruto, sem aplicações de regras de negócio, tratamento ou padronização. O processo seguiria algumas etapas para organização:

Padronização de Arquivos: Cada arquivo CSV seria nomeado com um sufixo correspondente à data do último registro contido nele, facilitando a identificação visual e o controle de carga, exemplo:  algodao01012026.csv


Organização e Particionamento:
O armazenamento seria organizado por tipo de commodity e estruturado através de partições temporais, com a data sendo criada a partir do sufixo do arquivo csv.
A hierarquia seguiria o seguinte modelo:
s3://[NOME-DO-BUCKET]/[CAMADA]/[PRODUTO]/year=YYYY/month=MM/day=DD/

Para acessar dados de algodão:
s3://bkt-datalake/raw/algodao/year=2026/month=01/day=01/algodao01012026.csv

Para acessar dados de soja:
s3://bkt-datalake/raw/soja/year=2026/month=01/day=01/soja01012026.csv

Uma vez com os dados persistido no S3, um gatilho (Event Bridge ou S3 Trigger) iniciaria automaticamente o pipeline de transformação. Esse fluxo seria responsável por processar e mover os dados para as camadas subsequentes: Processed (limpeza e tipagem) e Curated (aplicação de regras de negócio).
